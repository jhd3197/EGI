# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What EGI is

EGI (Emergencia ┬Ę Gente ┬Ę Info) is an offline-first, self-hostable system for family reunification after a disaster. People register others as `missing`, `found`, `safe`, `deceased`, `sighted`, or `care`; data is stored locally on the device first and synced to a community-run server when connectivity returns. A future Android app is planned for Bluetooth LE mesh sync between nearby phones.

The repo has three parts that are loosely coupled by an HTTP sync contract:
- `server/` ŌĆö Python + FastAPI + SQLite sync hub (the live, working backend).
- `frontend/` ŌĆö offline-first PWA prototype (the live, working client).
- `mobile/android/` ŌĆö planned/partially-scaffolded Android app (mostly direction docs).

User-facing strings are Spanish-first; code comments and English strings are the default per `CONTRIBUTING.md`.

## Server (`server/`)

FastAPI app, single SQLite table. This is where almost all real logic lives.

```bash
cd server
python -m venv .venv
.venv\Scripts\activate          # Windows;  source .venv/bin/activate on macOS/Linux
pip install -r requirements.txt
cp .env.example .env            # edit if needed
python -m db                    # initialize ./data/egi.db
uvicorn main:app --host 0.0.0.0 --port 3000 --reload
```

The server also serves the frontend: with it running, the full app is at `http://localhost:3000`. `python -m db` (i.e. `db.init_db()`) is idempotent and runs automatically on app startup too.

There is no test suite yet. The `npm test` line in `CONTRIBUTING.md` refers to the **deprecated** Node.js server in `server/_legacy_nodejs/` ŌĆö ignore it; the Python server is canonical.

### Architecture notes

- **`main.py`** is now just app wiring: middleware, `/uploads` mount, `/health`, `include_router()` calls, and the catch-all `GET /{path:path}` that serves the SPA and **must stay last** so API routes take precedence. It re-exports `ocr_image`/`extract_with_llm` and defines `UPLOAD_DIR` because the OCR import route reads those off `main` at request time (the surface the tests monkeypatch). Routes live in `routes/` (thin HTTP adapters: `persons.py`, `sync.py`, `imports.py`, `events.py` ŌĆö note `imports.py`, not `import.py`, since `import` is reserved), and business logic in `modules/` (`persons`, `reports`, `sync`, `events`, `ocr_import`). Pydantic models and the `validate_status`/`now_iso` helpers are centralized in **`models.py`**.
- **`db.py`** defines the schema inline in the `SCHEMA` string (there is **no separate `schema.sql` file** ŌĆö an open `schema.sql` buffer in the IDE is empty/non-existent). `get_db()` is a context manager opening a fresh connection per call with WAL mode. The valid `status` values are enforced both by a SQLite `CHECK` constraint here and by `validate_status()`/`VALID_STATUSES` in `models.py` ŌĆö keep the two lists in sync.
- **`ai.py`** is the local-first multi-provider AI base (`BaseExtractor`): default provider is **Ollama** (local, no API key) with **OpenAI** as an opt-in fallback, configured via `AI_PROVIDER`/`AI_MODEL`. `available()` cheaply checks reachability, `query()` returns raw text or `None`, and `parse_json()` robustly recovers JSON (strips `<think>` blocks and ``` fences, slices the object, drops trailing commas). It never raises and degrades to `None` so AI is always optional. Used by `modules/normalize.py` (the `POST /normalize` free-textŌåÆ`ai_draft` flow) and as the OCR fallback when `LLM_MODEL` is unset.
- **`research/synthetic.py`** generates fake person records for demos/load tests (`egi generate-synthetic`): internal name dictionary, c├®dulas in the reserved fake sub-range `V-00500000ŌĆ”` (offset from `egi seed`), weighted statuses, deterministic RNG so `--dry-run` matches a real run. Rows are `source='synthetic'` with a `TEST DATA` provenance, so `egi unseed` removes them.
- **`ocr.py`** handles the paper-import pipeline: `ocr_image()` preprocesses with Pillow then tries pytesseract, falling back to easyocr if installed; `extract_with_llm()` calls Prompture's `extract_with_model()` against the `ExtractedPaperReport` schema. If `LLM_MODEL` is unset, extraction is skipped and the OCR record holds raw text only.
- **Sync model:** `POST /sync` upserts keyed on the client-supplied `id` with **timestamp-guarded last-write-wins**: an incoming record is skipped when its `updatedAt` is older than the stored row's `updated_at` (ISO-8601 UTC compared lexicographically), so a stale mesh relay arriving out of order can't clobber a newer update. The response includes a `skipped` count. `GET /sync?since=ISO8601` returns records with `updated_at > since`. Clients own their record IDs and timestamps.
- **Moderation queue (`modules/moderation.py`, `routes/moderation.py`):** untrusted records (`source` in `ocr`/`ai_draft`/`pfif_import`, plus anything `reviewed=0`) await review. `reviewed` is the trust flag: `0` pending, `1` approved, `-1` rejected (soft-delete ŌĆö chosen over a `status='rejected'` value so the six valid statuses stay untouched). `GET /moderation/pending`, `POST /moderation/{id}/approve|reject`, `GET /moderation/stats`. The public `GET /persons` search applies the trust gate: it hides `reviewed=-1` and unreviewed untrusted-source rows, but keeps trusted web/seed rows (`reviewed=0`, `source='web'`) visible ŌĆö don't tighten this to "all reviewed=0" or normal web reports vanish.
- **OCR review flow:** `POST /import/paper` creates a draft with `source='ocr'` and `reviewed=0`. It stays out of the trusted set until a moderator calls `POST /import/paper/{id}/review` (which defaults `reviewed` to 1). Provenance is recorded so users can see which image a record came from.
- **Field name mismatch:** the API/JSON uses camelCase `createdAt`/`updatedAt` on `PersonRecord`, but the DB columns are snake_case `created_at`/`updated_at`. `main.py` maps between them explicitly in the sync INSERT ŌĆö watch this when adding fields.
- **Auth & RBAC (`auth.py`, `modules/users.py`, `routes/auth.py`, `routes/users.py`):** real user accounts replace static operator tokens (plan-08). `users`/`user_tokens` tables; passwords are bcrypt-hashed (cost 12) and tokens stored only as `SHA-256(token)` ŌĆö the raw token is shown once at login/creation. Roles are a strict hierarchy `viewer < operator < commander < admin`. The workhorse dependency is `require_role(min_role)`; `require_operator` is now a backward-compatible alias for `require_role("operator")`, so existing routes keep working. `require_role` accepts (in order) a valid user token meeting the role, a **deprecated** static `OPERATOR_TOKENS` token (operator-level only, logged), or ŌĆö if no users and no static tokens exist ŌĆö a dev bypass. `require_user` (no static/dev bypass) returns the full user dict for `/auth/me` etc. On first startup, if `OPERATOR_TOKENS` is set and no users exist, `main._bootstrap_admin()` seeds an admin and prints the password once. Audit actors are now real `user:<id>:<name>` principals. Manage accounts with `egi user ŌĆ”`.
- **Operational intelligence (plan-13):** dashboards, data-quality scoring and reporting, all over the existing tables plus two additive server-local tables (`data_quality_scores`, `scheduled_reports`). **Dashboards** (`modules/stats.py`, `routes/stats.py`, viewer-gated): `GET /stats/operations/{id}` (persons-by-status, daily intake, active action-plan tasks, geolocated/pending counts), `/stats/operations/{id}/timeseries`, `/stats/global` (cross-operation totals + the moderation queue and duplicate-cluster sizes). All counts exclude merged (`merged_into IS NOT NULL`) and rejected (`reviewed=-1`) rows; `moderation_queue` counts only untrusted-source `reviewed=0` rows (trusted web rows sit at `reviewed=0` but stay visible). **Data quality** (`modules/quality.py`, `routes/quality.py`): a 0-100 per-record score = weighted completeness/confidence/freshness with issue codes (`missing_name`, `unreviewed`, `stale`, `possible_duplicate`, ŌĆ”), cached in `data_quality_scores` (recomputable, never authoritative) ŌĆö `/quality/summary|low|stale|persons/{id}`, operator-gated `/quality/recalculate`; `egi quality-scan` is the nightly-job entry point (recalc + duplicate + stale report). **Suggested search sectors** (`geo.suggested_sectors`, `GET /operations/{id}/sectors`): grids geolocated persons+reports into density cells, returns the hottest as candidate sectors. **SITREP reports** (`modules/sitrep.py`): `GET /operations/{id}/sitrep?format=json|html|pdf` assembled from the dashboard aggregates; HTML/JSON always work, PDF degrades to 503 without reportlab (like the flyer). **Scheduled reports** (`modules/scheduled_reports.py`, `routes/reports.py`): `scheduled_reports` CRUD + a simple keyword schedule (`hourly`/`daily`/`weekly`), `run_due` generates + delivers each due SITREP via email (`modules.messaging`) or webhook id (`modules.webhooks.deliver`) and stamps `last_run_at`; driven by `POST /reports/run-due` or the `egi run-reports` cron CLI (`egi sitrep` prints an on-demand one). The PWA exposes an operator-gated `DashboardScreen.jsx` over `/stats`.
- **Interoperability & federation (plan-12):** four data-exchange surfaces, all operator-gated. **CSV/Excel** (`modules/exchange.py`, `routes/exchange.py`): `GET /export/persons.csv|.xlsx` (filter by status/disaster/since), `POST /import/persons` (multipart spreadsheet, es/en column-alias mapping, per-row validation, `auto_approve`/`dry_run`), `GET /import/persons/template.csv`. Bulk imports land as `source='csv_import'` (added to `UNTRUSTED_SOURCES`) so they await moderation. CSV uses stdlib `csv` (UTF-8 BOM); xlsx needs `openpyxl`. **PDF flyers** (`modules/flyer.py`): `GET /persons/{id}/flyer.pdf?lang=es|en|pt` ŌĆö reportlab + qrcode; embeds a photo only when `ENABLE_PHOTOS` is on; degrades to 503 if reportlab/qrcode are missing. **Webhooks** (`modules/webhooks.py`, `routes/webhooks.py`): subscription CRUD, HMAC-SHA256-signed JSON delivery over urllib, per-attempt `webhook_deliveries` log, exponential-backoff `retry_pending`. Events `person.created`/`person.updated` (emitted from `sync.sync_upload`), `person.merged` (`duplicates.merge_cluster`), `operation.closed` (`operations.close_operation`) ŌĆö emission is best-effort and post-commit via `webhooks.emit()`, wrapped so a webhook failure can never break the triggering request. **Federation** (`modules/federation.py`, `routes/federation.py`, `egi peer` CLI): `trusted_peers` with trust-on-first-use public-key pinning (changing a pinned key ŌåÆ 409); `pull_from_peer` feeds the peer's `/sync` response back through `sync.sync_upload` so the existing timestamp-guarded last-write-wins handles conflicts/dedup, `push_to_peer` posts local records since `last_push_at`. Webhook/federation HTTP uses stdlib urllib (no new runtime deps) and the module-level `_http_*` helpers are the surface tests monkeypatch.
- **Production operations & observability (plan-15):** **Health/metrics** ŌĆö `GET /health` (`modules/health.py`) is structured (database/uploads_dir/sync_queue checks; 503 when the database check fails ŌĆö only the DB is critical), `GET /ready` flips true after startup, `GET /metrics` (`metrics.py`) is hand-rolled Prometheus text (no new dep, same rationale as `ratelimit.py`): a request middleware in `main.py` records `egi_http_requests_total`/`egi_http_request_duration_seconds`, plus DB-derived gauges (db size, moderation/duplicate/message backlog, last backup) sampled at scrape time. **Privacy invariant:** metric labels use the route *template* (`route_template()`), never the concrete path, so no c├®dula/name/id leaks. **Logging** (`logging_config.py`): JSON formatter in production / text in dev (`LOG_FORMAT`), a `request_id` contextvar bound by the access middleware which echoes/generates `X-Request-ID` and logs one structured line per request (never tokens). **`GET /audit/log`** (`routes/audit.py`, operator-gated) merges `audit_log` + `record_history` with filters + `?format=export`. **Backups** (`backup.py`): `PRAGMA integrity_check` (aborts a corrupt backup, verified on restore), `prune_backups`, optional Fernet encryption + S3 (`boto3`) ŌĆö both degrade gracefully if the lib is absent; CLI `egi backup --retention-days/--encrypt/--s3-*`, `egi schedule-backup`, `egi restore` (decrypts `.enc`). **Migrations** (`migrate.py` + `server/migrations/NNNN_*.sql`, `schema_migrations` table): idempotent runner wired into `db.init_db`, `egi migrate [--check]` (CI-enforceable); `db.is_postgres()`/`database_url()` detect the opt-in `DATABASE_URL`; `egi sqlite-to-postgres` (`migrate_pg.py`, psycopg optional) is the **experimental** cutover ŌĆö SQLite stays the tested default (see `docs/POSTGRES.md`). **System events** (`modules/system_events.py`, `system_events` table, `GET /system/events|/system/migrations`) record what the server did to itself (startup, backup, migration) ŌĆö distinct from the audit log. **Security CI** (`.github/workflows/security.yml`, `bandit.yaml`, `.pre-commit-config.yaml`): pip-audit/npm-audit (advisory), bandit (blocking on HIGH; `B608` parameterized-SQL is a documented skip), gitleaks; `egi rotate-secrets` revokes all bearer tokens. Ops docs: `docs/OPERATIONS.md`, `docs/PERFORMANCE.md` (SLOs + `server/tests/load/locustfile.py`), `deploy/` (alert rules + staging compose). Two new server-local tables (`schema_migrations`, `system_events`) are additive; the version constant lives in `version.py`.

- **Shelter & refugee information hub (plan-20):** shelters are promoted from a frontend-only demo list into server-backed records. Four additive tables (`shelters`, `shelter_updates`, `shelter_checkins`, `shelter_tokens`) with `modules/shelters.py` + `routes/shelters.py`. `services`/`supply_needs`/`target_populations` are JSON-array TEXT columns decoded in the module; `trust` mirrors the moderation tiers (`official` > `volunteer` > `crowd`). **Reads are public** (`GET /shelters` with responder filters `has_space`/`accepts_pets`/`has_medical`/`needs_supplies`, `GET /shelters/{id}`, `/updates`, public family search `GET /shelters/checkins/search`). **Feed/capacity writes** (`POST /shelters/{id}/updates`, `PATCH /shelters/{id}/capacity`) accept an operator+ user OR the shelter's own verified operator (`_authorize_writer`); the feed `author_role` is decided server-side (always `official` for an authorized writer) so the badge can't be faked. **Check-in** (`POST /shelters/{id}/checkin`) is public + rate-limited. **Operator mode:** a commander mints one-time claim tokens (`egi shelter issue-token`, stored as `SHA-256` like user tokens), an operator redeems via `POST /shelters/claim` (sets `operator_user_id` + `trust='official'`), and gets a private roster + `GET /shelters/{id}/roster.csv`. Shelter upserts use the same timestamp-guarded last-write-wins as `/sync`. PWA: `ShelterDetailScreen.jsx` (capacity bar, service/supply/population chips, native turn-by-turn via `lib/directions.js` ŌåÆ `EgiBridge.openTurnByTurn`, update feed, check-in, inline operator panel) + tap-through/filters/claim in `SheltersScreen.jsx`.

- **Offline routing ŌĆö from X to Y (plan-21):** offline-first directions in the PWA + Android. Three additive server tables (`routing_packs`, `hazard_zones`, `route_shares`, `evacuation_corridors`; migrations `0002`ŌĆō`0005`) all server-local and loosely coupled. **Routing packs** (`modules/routing.py` + `routes/routing.py`, public `GET /routing/packs[/{id}[/meta]]`) are per-region road graphs (`{nodes:[[lat,lon]], edges:[[from,to,meters,flags]]}`) stored as JSON files under `server/data/routing_packs`; a deterministic La Guaira demo pack is seeded from `db.init_db`. The client (`lib/routePack.js`, IndexedDB `egi-routing`) downloads a pack and runs **A\*** in a Web Worker (`workers/routeWorker.js` over the pure `lib/routeGraph.js`) to follow roads; the polyline renders on `MapScreen.jsx`. **Always-on fallback** (`lib/directions.js`): Haversine distance + walking-time + compass bearing/`cardinalKey` step list, a 20-route history in IndexedDB `meta`, and `formatDistance` (km/mi). **Hazards** (`modules/hazards.py`, geometry = polygon or circle JSON, server-computed `bbox`): public `GET /hazards`, community `POST /hazards` lands `source='hazard_report'`/`reviewed=0` ŌåÆ moderation (operator `GET /hazards/pending` + `POST /hazards/{id}/review`); `lib/hazards.js` blocks A\* edges inside active hazards and the map overlays them. **Route shares** (`modules/route_shares.py`): `POST /routes/share` dedups near-identical shares within 6h (`dedup_key`), `GET /routes/shared`; PWA "share this route" + suggested-routes list ŌĆö BLE-direct propagation is deferred to the pending device BLE cert, so shares sync via the server bridge-node path today. **Multi-modal** (`lib/multimodal.js`): walk/drive/transit speeds, `estimateArrival` ranges, long-walk `batteryWarning`, and `hubToHub` two-leg evacuation routing through the nearest shelter; **evacuation corridors** (`modules/corridors.py`, operator-written, public `GET /corridors`) overlay the map. **Native Android** (`bridge/EgiBridge.kt` + `LocationCache.kt`): `getCurrentPosition()` returns a cached last-known fix (instant "my location") and `navigateTo`/`openTurnByTurn` hand off to a maps app; `MainActivity` grants WebView geolocation and runs the foreground-only location cache. New tables are additive; the SPA catch-all stays last; all new copy is in es/en/pt (+ partial guc ŌåÆ es).

### Config (`.env`)

`PORT`, `DB_PATH`, `UPLOAD_DIR`, `FRONTEND_DIR`, `TESSERACT_CMD`, and `LLM_MODEL` (format `provider/model`, e.g. `openai/gpt-4o-mini`). Provider keys come from standard env vars (`OPENAI_API_KEY`, etc.). OCR needs a Tesseract binary installed; point `TESSERACT_CMD` at it if not on PATH.

## Frontend (`frontend/`)

A **React + Vite** app. `npm install` then `npm run dev` (port 5173, proxies the API routes to the Python server on :3000); `npm run build` outputs `frontend/dist/`, which FastAPI serves in production. (It was rewritten from an earlier single-file `.dc`-template prototype that used a generated `support.js` runtime ŌĆö that prototype is gone.)

Architecture (state flows one way; components are presentational):
- **`src/store.js`** ŌĆö `useEgi()` hook holds *all* app state in one object with a `setState`-style merge helper, plus the offline cache/sync logic (`fetchAll`, `syncNow`, `queueRecord`) and every action. This is the port of the old `Component` class.
- **`src/lib/view.js`** ŌĆö `buildView(state, actions)` derives all display-ready values (decorated people, chips, connection styling, responsive flags) in one place. Ported from the prototype's `renderVals()`. Screens read from this `view` object and stay dumb.
- **`src/lib/css.js`** ŌĆö `css('padding:16px;ŌĆ”')` turns an inline CSS string into a React style object. The whole UI uses inline styles (carried over verbatim from the prototype); merge dynamic values with `{...css('ŌĆ”'), background: x}`.
- **`src/App.jsx`** routes between three states: `showAuth` ŌåÆ `AuthScreen`, `showPicker` ŌåÆ `DisasterPicker`, else `AppShell` (sidebar/topbar/tabbar + the five screens + `ReportSheet`).
- **Sync/offline:** same-origin relative API URLs by default; `localStorage.setItem('egi_api_url', ŌĆ”)` overrides for a remote server. On load it pulls `/sync` + `/persons`; offline it falls back to `localStorage`; new reports queue locally and `POST /sync` when back online.

Note: the prototype's `renderVals` never exposed `showAuth`, so the polished auth screen never rendered and reporting was unreachable for fresh users. The rewrite wires the intended flow (auth ŌåÆ picker ŌåÆ app), which is why `view.js` defines `showAuth/showPicker/showApp` coherently.

## `egi` CLI (`egi_cli/`)

A repo-root Click CLI (`pip install -e .` ŌåÆ `egi` command) wraps dev and data-ops tasks: `backend`, `frontend`, `build`, `seed`, `unseed`, `export-pfif`, `import-pfif`, `generate-synthetic`, `ocr-review`, and the `user` account-management group (`egi user create|list|passwd|set-role|activate|deactivate|delete`). It is a **dev/operator tool, kept out of the production PWA/APK**. The command group is in `egi_cli/cli.py`; each command is its own module in `egi_cli/commands/`. Command callbacks import server modules **lazily** (after `paths.ensure_server_importable()`, which also loads `server/.env` and pins `DB_PATH`/`UPLOAD_DIR` to `server/`), so the CLI always touches the same DB as a running `egi backend` no matter the CWD. Keep top-level imports in command modules to `click` only.

## Cross-cutting conventions

- New status values must be added in three places: the SQLite `CHECK` in `db.py`, `VALID_STATUSES` in `models.py`, and the `status` description in `ocr.py`'s `ExtractedPaperReport`.
- This data is sensitive (names, contacts, locations of people in a crisis). Follow the privacy principles in `README.md`: collect the minimum, mark unverified reports clearly, prefer corrections/history over silent deletion, and add no analytics/tracking.
